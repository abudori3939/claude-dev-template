## フェーズ粒度の方針（レビュー予算）

> 粒度は「実装者の都合」ではなく「**人間が1サイクルで追えるレビュー量**」に合わせる。迷ったら小さく。

- 較正レベル: **M（標準）** ← S / M / L から選ぶ。ユーザーに合わせて調整可。
  - **S（小）**: 1関数 / 1ふるまい・≤ ~80行・レビュー5分（細かく追いたい・学習中）
  - **M（標準）**: 1機能スライス・≤ ~200行・レビュー10〜15分
  - **L（大）**: 複数機能・≤ ~500行・レビュー30分（設計を任せたい上級者）
- 1サイクル（=1 PR）の単位: spec の「完了の目安」1つに対応する**縦スライス1本**。
- 新概念は1つ / 新規テスト 1〜3。超えそうなら**フェーズを分割**する。
- 着手ゲートで、差分規模が較正レベルに収まるかを一言確認する。
