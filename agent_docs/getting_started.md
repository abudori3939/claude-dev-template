# Getting Started（クローン後にまず読む）

このテンプレートを clone したら、実装に入る前に **フェーズ0: ドメイン設計** を行う。
目的は、作りたいソフトウェアの「目的・仕様」を固め、`agent_docs/project/plan.md` を練り上げること。

> エージェントは、セッション開始時に CLAUDE.md「セッション開始時の判定」でフェーズ0と判断したら、
> **実装に入らず本手順でユーザーの要件定義を主導する。**

## 役割分担（人間 / エージェント）

| 工程 | 人間（ユーザー） | 実装エージェント | レビューエージェント |
|------|------------------|------------------|----------------------|
| フェーズ0 ドメイン設計 | 目的・要件を提示し plan/spec を承認 | 対話で要件を引き出し plan/spec に整形 | — |
| 着手ゲート | 仕様サマリを見て Go 判断 | 最小サマリを提示 | — |
| レッド | テストの失敗を確認・承認 | 失敗するテストを書く | — |
| グリーン | — | 最小実装＋リファクタ | — |
| レビュー | 承認後、PR レビューに参加・マージ | 報告して停止 | PR にコメント |

> 実装とレビューは**別エージェント**。人間は「承認とマージ」の門番に徹し、監視コストを最小化する。

## フェーズ0の進め方（plan の練り上げ方）

要件は**一問ずつ**引き出し、ユーザーの負荷を上げない。確認は短く。

1. **目的**: 作りたいソフトウェア／コンポーネントが何をするかを 1〜3 行で確認 → `CLAUDE.md` プロジェクト概要 と `plan.md` 概要 に記入。

2. **使用スタックの選択**: エージェントは `agent_docs/stacks/` を確認し、ユーザーに次のように促す:

   > 「今回はどのスタックで開発を始めますか？
   > **(a) ROS 2 / C++**（`agent_docs/stacks/ros2_cpp.md`）
   > **(b) ROS 2 / Python**（`agent_docs/stacks/ros2_python.md`）
   > **(c) その他**（新しいスタックを追加して別環境で進める）」

   - (a)/(b) を選んだら、そのガイドに従って以降のビルド／テスト／lint を行う。
   - (c) の場合は、`agent_docs/stacks/<新stack>.md` を新規作成し、ビルド・テスト・lint・命名の作法を定義してから進む（既存ガイドを雛形にする）。
   - 選択結果は `CLAUDE.md` プロジェクト概要の「使用スタック」に記録する。

3. **入出力**: 主要な入力／出力インターフェース（トピック・API・データ・主要パラメータ）を確認。
4. **環境・制約**: 実行環境、外部依存、既知の課題を確認。
5. **機能分割と順序・粒度**: 機能を小さく分割し、実装順序（Phase）を決める → `plan.md` に反映。
   あわせて**フェーズ較正レベル（S / M / L、既定 M）**をユーザーに確認し、`plan.md`「フェーズ粒度の方針」に記録する（1サイクルを、人間がレビューで追える量に収める）。
6. **仕様化（Phase 0 で書くのは粗いレベルまで）**: `spec.md` を**2層で育てる**前提で、Phase 0 では次だけを確定する（書式・粒度の方針は `agent_docs/common/spec_format.md`）:
   - **システム全体の入出力**（外部境界）
   - **モジュール間インターフェース（境界の契約）**＝ Phase をまたいで受け渡すデータの公開 I/O
   - 各機能（plan の Phase と 1:1）の **目的 / 完了の目安**
   - → **関数レベルの入出力・振る舞いはここでは書かない。** それは各機能を実装する Phase の着手時（plan mode）に追記する。Phase 0 では関数レベルの欄をプレースホルダのまま残してよい。
7. **進捗初期化**: `progress.md` の現在地を「実装フェーズ・Phase 1」に更新し、最初のタスクを記入。
8. **リモート準備**: `agent_docs/common/remote_setup.md` に従い `main` のブランチ保護を設定。
9. 以降は `CLAUDE.md` の**着手ゲート**から TDD サイクルを回す。

## フェーズ0の完了条件（Definition of Done）

- [ ] `CLAUDE.md` のプロジェクト概要を記入し、テンプレ注記行を削除した
- [ ] 使用スタックを選択し、該当する `agent_docs/stacks/<stack>.md` を用意した
- [ ] フェーズ較正レベル（粒度）をユーザーと確認し `plan.md` に記録した
- [ ] `agent_docs/project/plan.md` を自プロジェクト用に書き換え、`<...>` プレースホルダが残っていない
- [ ] `agent_docs/project/spec.md` の **Phase 0 層を確定した**:「システム全体の入出力」「モジュール間インターフェース（境界の契約）」を実値で記入し、各機能（plan の Phase と 1:1）の **目的 / 完了の目安** を埋めた（これらの欄に `<...>` プレースホルダが残っていない）。関数レベル I/O は各 Phase 着手時に書くため未記入で可
- [ ] `agent_docs/project/progress.md` を実装フェーズに更新した
- [ ] `main` のブランチ保護を設定した（`agent_docs/common/remote_setup.md`）
- [ ] （任意）エンドユーザー向けドキュメントは `docs/` に別途用意する
